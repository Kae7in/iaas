ℤ Using Intz.io

Intz.io implements a simple RESTful API designed to easily allow you to store and retrieve your integers. Integers have the following format:
{
id: "77671624-9dfa-37d3-b459-abf9eae34317",
label: "my_integer",
value: 700
 }

You can query integers by id, label, value, or any combination therein. Consuming the Int.z API using cURL is simple:

GET a list all of your integers
curl -H http://dev.intz.io/integers "Authorization: Token AUTH_TOKEN"

GET an integer by id
curl -H http://dev.intz.io/integers/id "Authorization: Token AUTH_TOKEN"

GET a list of integers by value and/or label
curl -H http://dev.intz.io/integers -X GET -d "value=700" -d "label=my_integer" "Authorization: Token AUTH_TOKEN"

POST an integer with a value and/or label
curl -H http://dev.intz.io/integers -X POST -d "value=700" -d "label=my_integer" "Authorization: Token AUTH_TOKEN"

PUT to update an integer by id with a new value and/or label
curl -H http://dev.intz.io/integers/id -X PUT -d "value=700" -d "label=my_integer" "Authorization: Token AUTH_TOKEN"

PUT to update a list of integers filtered by value and/or label with newValue and/or newLabel
curl -H http://dev.intz.io/integers -X PUT -d "value=700" -d "label=my_integer" newValue=1000" -d "newLabel=her_integer "Authorization: Token AUTH_TOKEN"

DELETE an integer by id
curl -H http://dev.intz.io/integers/id -X DELETE "Authorization: Token AUTH_TOKEN"

DELETE a list of integers by value and/or label
curl -H http://dev.intz.io/integers -X DELETE value=700" -d "label=my_integer" "Authorization: Token AUTH_TOKEN"
